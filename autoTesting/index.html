    # 介绍

    单元测试是针对 程序的最小单元 来进行正确性检验的测试工作。程序单元是应用的最小可测试部件。一个单元可能是单个程序、类、对象、方法等。

    # 特性

    • 减少bug、提高代码质量、快速定位bug、减少调试时间、放心重构。
    • 不用在修改代码之后提心吊胆的等着浏览器刷新。
    • 修改一小部分代码，就可以模拟上下文，调试一个片段。
    • 运行速度也比编译整个项目快得多。
    • 如果一次没改好，也不用重复操作。
    • 测试用例当作调试用的。一边写代码，一边写测试用例。

    # 目的

    当你的项目足够大的时候，在叠加模块和组件的过程中，是很有可能影响之前的模块。但是被影响的模块已经通过了测试，我们在迭代的时候，很少有测试人员会去重新测试这个系统。所以， 被影响的模块很可能就有了一个隐形的bug被部署到线上。因此我们采用自动化测试。最主要的作用是对于大型项目，在每次迭代的时候， 可以保证整个系统的正确运行， 确保系统的健壮。

    # 测试工具

    使用vue官方推荐 Jest 和 @vue/test-utils 进行单元测试。
