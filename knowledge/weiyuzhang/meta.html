# viewport 简介

# 什么是 viewport

设备的屏幕上用来显示网页的区域

移动设备上的 viewport 就是设备的屏幕上能用来显示我们的网页的那一块区域，在具体一点，就是浏览器上(也可能是一个 app 中的 webview)用来显示网页的那部分区域。

但 viewport 又不局限于浏览器可视区域的大小，它可能比浏览器的可视区域要大，也可能比浏览器的可视区域要小。

通用设备上浏览器的默认 viewport 的宽度



# CSS 中的 1px 不等于设备的 1px

先看不同移动设备的像素、倍率：

  • 在桌面浏览器中二者是相等的
  • 在手机端根据 PPI 的不同，倍图也是不相同
  • 在用户对显示部分进行缩放

# 三个 viewport 理论

# 1、Layout viewport -- 浏览器窗口默认的 viewport（布局视口）

网页布局的区域，它是<\html>元素的父容器，通过 document.documentElement.clientWidth 来获取。

  • Layout viewport 用 CSS 像素来衡量尺寸，在缩放、调整浏览器窗口的时候不会改变。
  • 在桌面浏览器中，缩放为100%的时候，Layout viewport 宽度等于内容窗口的宽度。

在移动端，缩放为100%的情况下，Layout Viewport 不一定等于内容窗口的大小。当你用手机浏览浏览宽大的网页（这些网页没有采用响应 式设计）的时候，你只能一次浏览网页的一个部分。



# 2、visual viewport – 浏览器可视区域（视觉视口）

  • 缩放、调整浏览器窗口改变的只是 visual viewport。
  • visual viewport 用 css 像素来衡量尺寸，表示有多少个 css 像素能够被用户看到。
  • 通过 window.innerWidth 获取

# 3、ideal viewport – 移动设备的理想 viewport（理想视口）

  • 显示的文字大小合适
  • ideal viewport 的宽度等于移动设备的屏幕宽度
  • 不需要用户缩放以及拉取横向滚动条可以查看到网站的所有内容
  • 无论在何种分辨率的屏幕下，那些针对 ideal viewport 而设计的网站，不需要用户手动缩放，也不需要出现横向滚动条，都可以完美的呈现给用户。

# 设置 viewport

一个常用的针对移动网页优化过的页面的 viewport meta 标签大致如下：

  <meta name="viewport" content="width=device-width, initial-scale=1.0">
1

# 属性

  • width：控制 viewport 的大小，可以指定的一个值，如 600，或者特殊的值，如 device-width 为设备的宽度（单位为缩放为 100% 时的 CSS 的像素）。
  • height：和 width 相对应，指定高度。
  • initial-scale：初始缩放比例，也即是当页面第一次 load 的时候缩放比例。
  • maximum-scale：允许用户缩放到的最大比例。
  • minimum-scale：允许用户缩放到的最小比例。
  • user-scalable：用户是否可以手动缩放。

# 设置 initial-scale 的影响

缩放系数 = 理想视口宽度（ideal viewport） / 视觉视口宽度（visual viewport）

如果设置了 initial-scale 比如为0.5，那么以 iPhone6 为例，iPhone6 的设备宽度是375px，即理想视口宽度也为375px，所以视觉视口宽度 = 375px（理想视口宽度）/ 0.5（缩放系数）。很明显设置了 initial-scale 就相当于初始化了视觉视口，而且会将布局视口初始化为这个视觉视口的值。

# width 和 initial-scale 共存

width用于指定布局视口的大小，那么一起设置的话听谁的呢？

还是以 iPhone6 为例，它的尺寸是667（h） * 375（w），对其进行如下设置

  <font class="red"><meta name="viewport" content="width=400, initial-scale=1">
1

在手机横屏和手机竖屏两种情况下执行

  console.log(布局视口: ${document.documentElement.clientWidth});
  onsole.log(视觉视口: ${window.innerWidth});
1
2

竖屏 375 * 667：会得到“布局视口: 400; 视觉视口: 400”。

横屏 667 * 375：会得到“布局视口: 667; 视觉视口: 667”。

结论是：width 与 initial-scale都会初始化布局视口，但浏览器会取其最大值。

# 设置理想视口 ideal viewport

这时候再看这行代码：

  <meta name="viewport" content="width=device-width, initial-scale=1">
1

明明 width = device-width 和 initial-scale = 1 都是去初始化布局视口成理想的布局视口，只写其中一个就可以了，为什么要两个都一起写呢？

因为有的浏览器只设置其中一个，不能保证理想视口的尺寸能随着屏幕的旋转而正确改变，所以两个一起写只是为了解决兼容性问题。

# 利用 meta 标签控制 viewport

# 使用 document.write 来动态输出 meta 标签

  document.write("<meta name='viewport' content>")

# 使用 setAttribute 来改变

  let port = document.getElementById('testViewport'); 
  port.setAttribute('content', 'width=480');
