# safe-area-inset-*

在设置 viewport-fit=cover 之后，Web中会新增四个常量。

在 iOS 11中的 WebKit 包含了一个新的 CSS 函数 constant()，以及一组四个预定义的常量：safe-area-inset-left、safe-area-inset-right、safe-area-inset-top 和 safe-area-inset-bottom。当合并一起使用时，允许样式引用每个方面的安全区域的大小。

  • safe-area-inset-top：导航栏 + 状态栏的高度
  • safe-area-inset-right：如果未竖屏时为 0
  • safe-area-inset-left：底下圆弧的高度
  • safe-area-inset-bottom：如果未竖屏时为 0

分别表示 safe area 和可视窗口 viewport 顶部，右边，左边，底部的间距，可以用于设置 margin 和 padding 或者绝对定位时 left 和 top。

当我们设置 viewport-fit: contain，也就是默认的时候，设置 safe-area-inset-left, safe-area-inset-right, safe-area-inset-top 和 safe-area-inset-bottom 等参数时不起作用的。

当我们设置 viewport-fit:cover 时：设置如下：

  body {
    padding-top: constant(safe-area-inset-top);
    padding-right: constant(safe-area-inset-right);
    padding-bottom: constant(safe-area-inset-bottom);
    padding-left: constant(safe-area-inset-left);
  }
